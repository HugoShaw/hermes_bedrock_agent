You are working on a document parsing experiment for Japanese enterprise-style Excel design documents.

Japanese companies often use Excel files as software design documents. These Excel files may contain business process diagrams, system flow diagrams, data transformation rules, interface definitions, database table mappings, field-level mappings, and conversion logic between multiple systems.

Each Excel workbook usually contains multiple sheets. The first sheet is often an overview or document introduction, while the following sheets usually contain the actual business logic, flowcharts, mapping tables, field definitions, and detailed system knowledge.

Your task is to parse each Excel workbook sheet by sheet and convert the visual and tabular knowledge into Markdown and Mermaid text.

## Main Requirements

1. Process each Excel workbook independently.

2. Process every sheet in each workbook independently.

3. For each sheet, first convert the sheet into image screenshots.

4. If a sheet is too large, too wide, or too tall to be captured completely in one screenshot, split the sheet into multiple image tiles or regions.

5. After capturing all image tiles for a single sheet, use code to stitch those tiles back into one complete full-sheet image.

6. The stitched image must preserve the original sheet layout, relative positions, visual structure, and as much original resolution as possible.

7. Send the full-sheet image to Claude Sonnet multimodal model for visual understanding.

8. Ask Claude to analyze the full-sheet image and extract the following knowledge:
   - Overall sheet purpose
   - Important tables
   - Field mappings
   - Source and target systems
   - Source and target tables
   - Source and target fields
   - Data transformation rules
   - Business rules
   - Process flow logic
   - Flowchart nodes and edges
   - Conditions, branches, labels, and annotations
   - Any relationships between systems, tables, fields, APIs, files, and processes

9. Generate a Markdown file for each sheet.

10. If the sheet contains a flowchart, generate a Mermaid diagram that represents the flowchart logic.

11. If the sheet contains mapping tables, extract the mappings into clean Markdown tables.

12. If the sheet contains both flowcharts and tables, combine both types of knowledge into one structured Markdown output.

13. Preserve Japanese text exactly as much as possible. Do not translate Japanese terms unless explicitly required.

14. Do not rely on hard-coded sheet names, fixed cell positions, or company-specific keywords. The solution must be general enough to handle different Japanese enterprise Excel design documents.

15. The final output should be useful for building a GraphRAG knowledge base later.

## Expected Output

For each Excel workbook, generate a structured output folder.

For each sheet, generate:

- `sheet_name.md`  
  A Markdown document describing the sheet content, extracted tables, mappings, business rules, and process logic.

- `sheet_name.mmd`  
  A Mermaid diagram file if the sheet contains a flowchart or process diagram.

- `sheet_name_full.png`  
  The stitched full-sheet image used for visual analysis.

- Optional intermediate image tiles for audit purposes.

## Markdown Output Structure

Each sheet-level Markdown file should follow this structure:

```markdown
# Sheet: <sheet name>

## 1. Sheet Overview

Describe the purpose of this sheet.

## 2. Visual Structure

Describe the main visual layout, including tables, flowcharts, sections, arrows, labels, and grouped areas.

## 3. Extracted Tables

Extract important tables as Markdown tables.

## 4. Field Mapping / Data Mapping

If the sheet contains mapping information, extract it in a structured table.

| Source System | Source Table | Source Field | Target System | Target Table | Target Field | Transformation Rule | Notes |
|---|---|---|---|---|---|---|---|

## 5. Business Rules

List business rules, conditions, filters, validation rules, and transformation logic.

## 6. Flowchart Explanation

If the sheet contains a flowchart, explain the flow in natural language.

## 7. Mermaid Diagram

If applicable, include the Mermaid diagram.

## 8. Uncertain or Ambiguous Points

List any parts that are visually unclear, truncated, ambiguous, or require manual review.
````

## Mermaid Requirements

When generating Mermaid diagrams:

* Use `flowchart TD` or `flowchart LR` depending on the original diagram layout.
* Preserve the original flow direction as much as possible.
* Preserve node labels in Japanese.
* Preserve branch labels and connector labels.
* Do not invent missing nodes or edges.
* If a connection is uncertain, mark it clearly in the Markdown explanation.
* Keep the Mermaid syntax valid.

## Important Notes

The goal is not only OCR. The goal is to understand the design knowledge inside the Excel sheets.

The parser should combine:

* Excel structural information
* Cell values
* Sheet dimensions
* Screenshot-based visual understanding
* Shape and arrow layout
* Claude Sonnet multimodal reasoning

The final result should be a high-quality Markdown and Mermaid representation of the original Excel design documents, suitable for manual review and later ingestion into a GraphRAG knowledge base.
